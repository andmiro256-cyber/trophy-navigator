## Раздел 1 — ТОЧКИ (Waypoints)

### 1.1 Добавление точки
- Кнопка в тулбаре → курсор становится перекрестьем
- Клик на карту → мгновенно ставится точка с именем по умолчанию (КП-01, КП-02...)
- Маленький попап рядом с точкой для быстрого переименования (не модальное окно)
- ПКМ на точке → полные свойства

### 1.2 Свойства точки
| Поле | Описание |
|------|----------|
| Название | До 16 символов, по умолчанию wp1/wp2/... |
| Описание | До 255 символов |
| Иконка | Выбор из набора (~30-50 символов), по умолчанию кружок |
| Цвет | Цвет маркера |
| Радиус сближения (Proximity) | Визуальный круг на карте. 0 = не показывать. По умолчанию берётся из глобальных настроек, но можно переопределить для каждой точки |
| Координаты | Отображение в формате °MM.mmm' + ручной ввод |

> **Тип точки (КП/Старт/Финиш) — не нужен.** Достаточно иконки и названия.

### 1.3 Именование точек
- По умолчанию: **wp1, wp2, wp3...** (латиница, как в OziExplorer/Garmin)
- Возможность переименовать в любой момент (двойной клик по имени или через свойства)

### 1.4 Список точек (Waypoint List) — отдельное окно
- Открывается кнопкой в тулбаре или меню, не боковая панель
- Таблица: №, Иконка, Название, Координаты, Радиус
- Сортировка по любому столбцу
- Поиск/фильтр по названию
- Выделение нескольких строк (Shift/Ctrl+клик)
- Кнопки: Центрировать на карте / Свойства / Удалить / Выделить все
- Массовое редактирование выделенных (радиус, цвет, иконка)
- Drag-and-drop для изменения порядка

### 1.4 Иконки точек
- Набор символов по образцу Garmin waypoint symbols (~30-50 иконок)
- По умолчанию: кружок
- Примеры: кружок, флажок, квадрат, ромб, треугольник, звезда, крест, дом, машина, палатка, топливо, вода, опасность, финиш и т.д.
- Выбор иконки в свойствах точки (визуальная сетка)
- Размер иконок — **глобальная настройка** (как шрифт, влияет на все точки сразу)
- Цвет — задаётся для каждой точки отдельно в свойствах

### 1.5 Форматы импорт/экспорт
- OziExplorer .wpt
- GPX waypoints
- CSV (lat,lon,name,desc)

## Раздел 2 — ТРЕКИ (Tracks)

### 2.1 Ручное рисование трека
- Кнопка в тулбаре → режим рисования
- Клик = добавить узел, линия рисуется в реальном времени
- **Перетаскивание узла:** зажал мышью по точке → тащишь в новое место (работает сразу, без отдельного режима)
- **ПКМ по узлу → удалить узел**
- ПКМ по линии между узлами → добавить узел в этом месте
- Alt+клик = начать новую секцию (разрыв)
- ESC или кнопка "Завершить" = зафиксировать трек

### 2.2 Редактирование треков (и нарисованных, и загруженных из файла — одинаково)
- **Продолжить трек** — добавить новые узлы в конец существующего
- **Перетаскивание узла** — зажал мышью → тащишь
- **ПКМ по узлу** → удалить узел
- **ПКМ по линии** → добавить узел в этом месте
- **Split** — разбить трек на два в выбранном узле
- **Join** — объединить два трека в один
- **Reverse** — перевернуть направление
- **Simplify** — упростить (убрать лишние точки)
- Загруженный .plt или .gpx трек редактируется так же как нарисованный вручную

### 2.3 Свойства трека
| Поле | Описание |
|------|----------|
| Название | Строка |
| Цвет линии | Выбор цвета |
| Толщина линии | **Глобальная настройка** (настроил — применяется везде) |
| Количество точек | Инфо |
| **Длина** | **км — отображается в панели треков** |
| Источник | ручной / файл |

### 2.4 Инструменты редактирования трека
- **Выделение участка:** мышью обвести → удалить/вырезать
- **Split:** разбить трек на два в выбранной точке
- **Join:** объединить два трека
- **Reverse:** перевернуть направление
- **Filter/Simplify:** упростить трек (убрать лишние точки, Douglas-Peucker)
- **Профиль высот:** график высоты по длине трека (если есть данные)

### 2.5 Список треков
- Список с цветным индикатором, названием, длиной
- Показать/скрыть трек (checkbox)
- ПКМ → свойства / инструменты
- Drag-and-drop порядка

### 2.6 Форматы
- GPX tracks
- OziExplorer .plt
- KML

## Раздел 3 — МАРШРУТЫ (Routes)

> Маршрут ≠ Трек. Маршрут — это список ТОЧЕК в заданном порядке.
> Трек — это набор GPS-координат (линия).

### 3.1 Создание маршрута
- Кнопка "Новый маршрут" → ввести название
- Цвет назначается автоматически (из набора), можно сменить вручную
- Режим добавления точек: кликаешь по маркерам точек на карте → они добавляются
- Или выбираешь из списка точек (Waypoint List)
- Маршрут отображается пунктирной линией, соединяющей точки по порядку
- Стрелки направления на линии

### 3.2 Редактирование маршрута
- Список точек маршрута с кнопками ↑ ↓ для изменения порядка
- Удалить точку из маршрута (точка остаётся на карте)
- Вставить точку между двумя существующими
- Переименовать маршрут

### 3.3 Информация о маршруте
- **Общая длина маршрута — отображается в панели маршрутов**
- Расстояние между соседними КП
- Азимут между КП

### 3.4 Примечания
- Не все КП на карте входят в маршрут — это нормально
- Маршрут = только выбранные КП в нужном порядке
- Максимум 20 маршрутов на одной карте одновременно
- Программа для ПЛАНИРОВАНИЯ, не навигатор — GPS не нужен

### 3.5 Форматы
- GPX routes
- OziExplorer .rte
- Экспорт в PDF (схема маршрута)

### 4.3 Управление слоями (в тулбаре, кнопка "Слои")
- **Базовая карта** — radio-список, одна активна
- **Оверлеи** — список с checkbox и ползунком прозрачности для каждого
  - По умолчанию: **100% (непрозрачный)**. Пользователь может убавить до 0%
  - Ползунок сохраняется между сессиями для каждого слоя
- Ползунок важен: спутник + Яндекс дороги на 70% = идеальная офроадная карта

### 4.4 Скачивание карт для офлайн (RMaps/Locus .sqlitedb)

#### Концепция
1. Пользователь активирует инструмент "Скачать карту" (кнопка в тулбаре)
2. Выделяет область — **прямоугольник** или **произвольный полигон** (кликами, двойной клик = завершить)
3. В диалоге выбирает: какой слой(и) скачать + диапазон зумов (min/max)
4. Приложение скачивает тайлы через Rust backend (без CORS ограничений) и сохраняет в .sqlitedb
5. Прогресс-бар: X из N тайлов, скорость, оставшееся время

#### Формат RMaps SQLite (.sqlitedb)
```sql
CREATE TABLE tiles (
  x INTEGER, y INTEGER, z INTEGER, s INTEGER,
  image BLOB,
  PRIMARY KEY (x, y, z, s)
);
CREATE TABLE info (minzoom INTEGER, maxzoom INTEGER);
-- z хранится как: 17 - zoom_level (инвертировано)
-- s всегда 0
-- image — PNG/JPEG байты тайла
```

#### Сохранение нескольких слоёв
- Каждый слой → отдельный .sqlitedb файл (как принято в Locus/RMaps)
- Имя файла: `ГГ-КК_Слой_z8-14.sqlitedb` (дата + название слоя + зумы)
- Можно скачать несколько слоёв за раз (очередь)

#### Оценка объёма
- Перед скачиванием показывать: ~N тайлов, ~X МБ (примерно)
- Тайлов = сумма по всем зумам: `4^(z - minZ) * tileCount(minZ)`
- Ограничение: не более 50 000 тайлов за одно скачивание (с предупреждением)

#### Технически (Rust backend)
- `reqwest` — HTTP клиент для загрузки тайлов
- `rusqlite` — запись в SQLite
- Параллельная загрузка: 4-8 потоков (настраивается)
- Respect rate limits: задержка между запросами к одному серверу
- Tauri event → прогресс в UI

#### Инструменты выделения (Leaflet.draw)
- **Прямоугольник** — зажал и нарисовал bbox
- **Полигон** — клики по вершинам, двойной клик = замкнуть
- Выделение сохраняется на карте, можно отредактировать перед скачиванием
- ПКМ на выделении → удалить / скачать

## Раздел 6 — ИНТЕРФЕЙС

**Принцип:** карта занимает максимум площади. Боковых панелей нет. Всё управление через верхнюю панель и всплывающие окна.

### 6.1 Верхняя панель (тулбар)

```
[⚙ Настройки] | [Карта▼] | [✋Рука] | [• Точка] [≡ Точки] | [〜 Трек] [≡ Треки] | [⚑ Маршрут] [≡ Маршруты] | [📐 Линейка] [⬇ Скачать]
```

**Секция 1 — Настройки и карта:**
- **⚙ Настройки** — ПЕРВАЯ кнопка. Открывает окно глобальных настроек программы
- **Карта▼** — выпадающий список: базовые карты + оверлеи с ползунком прозрачности

**Секция 2 — Режим курсора:**
- **✋ Рука** — обычный режим навигации (по умолчанию, пробел = временно переключить)

**Секция 3 — Точки:**
- **• Точка** — режим рисования: клик на карту = поставить точку
- **≡ Точки** — открыть окно "Список точек"

**Секция 4 — Треки:**
- **〜 Трек** — режим рисования: клики создают трек, ESC = завершить
- **≡ Треки** — открыть окно "Список треков"

**Секция 5 — Маршруты:**
- **⚑ Маршрут** — режим создания: кликаешь по существующим КП на карте → они соединяются линией маршрута
- **≡ Маршруты** — открыть окно "Список маршрутов"

**Секция 6 — Инструменты:**
- **📐 Линейка** — кликаешь точки, показывает расстояние сегментов и суммарное
- **⬇ Скачать** — режим выделения области (прямоугольник / полигон) → скачать тайлы в .sqlitedb

### 6.4 Окно "Список треков" (открывается кнопкой ≡ Треки)

- **ПКМ по строке** → Свойства / Центрировать / Удалить
- В окне: **ползунок цвета** и **ползунок толщины** для выбранного трека
- Кнопки: [Выделить все] [Удалить выделенные]

### 6.5 Окно "Список маршрутов" (открывается кнопкой ≡ Маршруты)
- Список маршрутов: название, кол-во КП, суммарная длина

### 6.6 Карта
- Занимает весь экран под тулбаром
- Никаких постоянных панелей по бокам

### 6.7 Попап на объекте (клик левой кнопкой)
- **Клик на точку** → маленький попап: название, координаты, [Свойства] [Удалить]
- **Клик на трек** → попап: название, длина, [Свойства] [Удалить]
- **ПКМ** на любом объекте → контекстное меню

### 6.5 Строка состояния (снизу)

## Раздел 7 — ФАЙЛЫ И ПРОЕКТ

### 7.1 Хранение файлов (как в OziExplorer — раздельно)
- Точки → .wpt (OziExplorer) или .gpx
- Треки → .plt (OziExplorer) или .gpx
- Маршруты → .rte (OziExplorer) или .gpx
- Каждый тип открывается и сохраняется отдельно

### 7.2 Импорт
- OziExplorer: .wpt, .plt, .rte
- GPX — всё из одного файла или выборочно (только точки / только треки / только маршруты)

### 7.3 Экспорт
- OziExplorer: .wpt, .plt, .rte (раздельно)
- GPX: либо всё вместе, либо раздельно по выбору
- Копировать координаты точки в буфер обмена в формате `55°45.132'N  37°36.934'E`
